## 1. Концептуальная архитектура статьи

Статья построена как сопряжение **четырёх теоретических источников** в петербургско-московской гуманитарной традиции:

1. **Топоров 2006** — сборник «Текст и комментарий» к 75-летию Вяч. Вс. Иванова, М.: Наука. Постановка проблемы жанра комментария в общефилологическом измерении.
2. **Парибок 2007 / 2011** — две статьи: краткая (СПбГУ 2007, через цитаты Лидовой) + развёрнутая («Шабдапракаша» 2011, с. 80–86, полный текст у нас в `/tmp/paribok_full.txt`). Из неё выведена **четвёртая ось** анализа: *понятие / концепт / кодификатор направления деятельности*. NB: в косвенных падежах правильно «Парибка», не «Парибока».
3. **Лидова 2024** — «Комментарий в Древней Индии» в сборнике ИМЛИ. Из неё выведена **третья ось**: пять *lakṣaṇa* объяснения по «Парашара-упапуране» (*pada-ccheda*, *padārtha*, *vigraha*, *vākya-yojanā*, *ākṣepa-samādhāna*).
4. **Казанский 2025** — статья в ИЕЯКФ 29, DOI 10.30842/ielcp2306901529049. Из неё выведена **вторая ось**: четыре типа комментария (филологический A / реалийный B / исторический V / культурологический G) + пять сквозных параметров переводческой стратегии.

К ним добавлены **дополнительные статьи из «Шабдапракаши» 2011 г.** (Зографский сборник I, СПб., под ред. Я. В. Василькова и С. В. Пахомова):
- **Лелюхин 2011** (с. 55–64) — параллельный кейс *adhyakṣa* в «Артхашастре»: историко-социологический пример «ложных друзей переводчика».
- **Скороходова 2011** (с. 267–278) — реформаторская традиция Бенгальского Возрождения как третья комментаторская линия наряду с *bhāṣya* и западной индологией.

**Центральный тезис статьи** (формулировка пользователя, утверждена в диалоге): русский переводчик санскритского текста наследует *две* комментаторские традиции одновременно — индийскую (где комментарий есть условие читаемости текста) и европейскую филологическую (где комментарий — вспомогательный аппарат). Стратегии русских санскритологов — разные конфигурации этого *двойного долга*.

## 5. Методология работы (как это было сделано)

### 5.1. Принципы общения с пользователем
1. **Не задавать вопросов больше одного-двух за раз.** Пользователь явно сформулировал, что предпочитает плотный поток уточняющих вопросов малыми порциями.
2. **Перед каждой развилкой — спрашивать.** Не делать предположения о том, какой из вариантов решения предпочесть, если есть содержательная разница.
3. **Чётко различать два типа предложений:** что уже написано / что я предлагаю как методологический шаг и жду санкции.
4. **Не льстить руководителю конференции.** Имя Казанского не должно фигурировать в заголовке статьи или быть избыточно цитировано — это будет воспринято как лесть, поскольку он сам автор близкой статьи и руководитель чтений.
5. **Не выдумывать факты, которые могут быть проверены.** Если статус издания неизвестен — спрашивать пользователя, не заменять предположением. Особенно это касается Леонова: тома 2022 г. не существует, любое такое упоминание — ошибка.
6. **«Парибка» — единственно корректная форма родительного и винительного падежей.** Не «Парибока». Это пожелание пользователя.

### 5.2. Технические принципы
1. **Парсить файлы через Python в `/tmp/` или `/home/claude/`.** Для PDF — `pdftotext -layout`. Для HTML — `re.findall(r'class="([^"]+)"', html)` для обнаружения структуры.
2. **Финальный документ собирать инкрементно через str_replace.** Не переписывать целиком: каждое изменение — точечная замена с проверкой `assert old in text`.
3. **После каждой правки — verification pass:** искать запрещённые формулы («М.: Наука, 2022», «том 2022»), проверять присутствие ключевых терминов.
4. **Промежуточные файлы хранить в `/home/claude/`, финальные — в `/mnt/user-data/outputs/`.** В конце сборки удалять промежуточные `.md` из outputs, чтобы пользователь видел только итоговый артефакт.
5. **Перед запуском глубокого исследования — задавать уточнения о приоритетах.** Глубинное исследование (extended search task) даёт большой массив данных; без приоритетов он становится плохо применимым.

### 5.3. Стилистика статьи
1. **Прямой профессиональный тон.** Без эмоций, без лести, без риторических вопросов в основном тексте.
2. **Структурное цитирование.** Каждая важная цитата — с указанием источника и страницы; формулы вида «(Парибок 2011, с. 86)» в основном тексте.
3. **Параграфы в 4–8 предложений.** Длинные параграфы дробятся.
4. **Ноль bullet-точек в основном тексте.** Списки — только в Приложениях, в чек-листе §9 и в табличных сводках §4.
5. **Курсив** — для санскритских терминов (`*dharma*`, `*ātman*`); **полужирный** — только для категориальных меток типа «**A** филологический» или для центральных тезисов.

## 7. Чего стоит избегать

1. **Не вводить дополнительные оси без санкции пользователя.** Сейчас их 4; пятая (например, «дискурсивный регистр», «институциональная привязка», «время написания примечания») — соблазн появится, но это уже усложнит модель за полезный предел.
2. **Не упоминать Лосева-монографа.** Пользователь явно сказал: для Платоновского восьмитомника берётся только Лосев-как-редактор-примечаний (1968–1972). Лосев-монограф откладывается до Топорова.
3. **Не переходить с темы статьи в тему репозитория или сайта.** Пользователь явно разделил уровни: статья → главное; репозиторий → надстройка над корпусом; samskrtam.ru → инфраструктура. Каждый уровень имеет свой формат.
4. **Не объединять «комментаторскую традицию» и «школу перевода» в одну категорию.** В Индии — традиции; в России — школы. Это разные жанровые конвенции.
5. **Не цитировать пять традиционных комментаторов Рамаяны без сверки сканов.** На текущий момент все упоминания Tilaka/Bhūṣaṇa/Śiromaṇi/Tattvadīpikā/Amṛta — рабочие гипотезы пользователя, не верифицированные атрибуции.
6. **Не использовать formatting-перегруз.** Bullet-точки в основном тексте, длинные таблицы во введении, эмодзи, заголовки уровня h4 — всё это снижает академичность.
